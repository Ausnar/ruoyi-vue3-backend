# 数据库变更流程约定

> 文档状态：当前有效 / 权威版本
>
> 使用说明：
> - 本文是数据库变更流程的权威版本。
> - `docs/数据库变更流程入口.md` 仅作为 `docs/` 目录下的兼容入口保留。
> - 当前默认开发、验证和交接基线是正式库 `ruoyi`；`ruoyi_ai_dev` 作为显式维护的保底 / 样例库保留。

## 目标

本项目的数据库变更采用“正式库 `ruoyi` 验证 + dev/prod 双脚本留档”的策略。涉及历史 dev -> 正式库迁移时，默认优先使用 Navicat `数据传输`，不默认假设跨库 SQL 一定可执行。

## 当前环境

- 当前开发与验证基线：`ruoyi`
- 显式维护的保底 / 样例库：`ruoyi_ai_dev`

## 固定规则

1. 所有面向当前系统事实的结构调整、初始化脚本和验证，默认以正式库 `ruoyi` 为基线。
2. `ruoyi_ai_dev` 作为保底 / 样例库参考；除非本轮已明确同步，否则不能用它推断当前默认开发状态。
3. 发往正式库 `ruoyi` 的 SQL 只能包含：
   - 最终确认的结构变更
   - 必要的正式初始化数据
4. 发往正式库 `ruoyi` 的 SQL 不能包含：
   - 测试会话数据
   - 临时联调数据
   - 仅用于开发验证的脏数据
5. 任何涉及数据库结构、菜单、权限或初始化数据的需求，默认输出两类脚本：
   - `dev` 联调脚本
   - `prod` 正式上库脚本
6. 如果需要从 dev 补齐正式库，默认优先走 Navicat `数据传输`；跨库 SQL 只有在权限和环境明确可行时才使用。
7. Quartz 自动同步任务在正式库 `ruoyi` 需要手工确认和维护，不默认从 `ruoyi_ai_dev` 自动迁入。
8. 每轮数据库结构收口后，需要明确记录 `ruoyi_ai_dev` 是否同步；如作为保底库使用，应执行对应 dev 脚本或通过 Navicat 显式同步，并刷新 `docs/ruoyi_ai_dev数据库结构说明.md`。

## SQL 目录约定

- 开发联调 SQL：`sql/dev/`
- 正式上库 SQL：`sql/prod/`

推荐命名方式：

- `sql/dev/20260402_xxx_dev.sql`
- `sql/prod/20260402_xxx_prod.sql`

## 执行责任

- 开发阶段：以正式库 `ruoyi` 当前结构和数据为默认验证基线，必要时可另开样例库验证风险脚本。
- 正式阶段：由人工审核后，将 `sql/prod/` 下脚本执行到 `ruoyi`，并同步刷新结构快照和业务语义文档。

## 说明

本约定用于后续所有数据库相关改造，除非用户明确指定历史 dev 联调，否则默认遵守本流程。
