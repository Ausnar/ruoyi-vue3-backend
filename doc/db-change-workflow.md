# 数据库变更流程约定

## 目标

本项目的数据库变更采用开发库验证、正式库人工执行的双脚本策略。

## 当前环境

- 开发联调库：`ruoyi_ai_dev`
- 正式业务库：`ruoyi`

## 固定规则

1. 所有数据库结构调整、初始化脚本、测试数据注入，先在 `ruoyi_ai_dev` 中执行。
2. 所有功能必须先在 `ruoyi_ai_dev` 中联调通过，再整理正式上库脚本。
3. 发往正式库 `ruoyi` 的 SQL 只能包含：
   - 最终确认的结构变更
   - 必要的正式初始化数据
4. 发往正式库 `ruoyi` 的 SQL 不能包含：
   - 测试会话数据
   - 临时联调数据
   - 仅用于开发验证的脏数据
5. 任何涉及数据库的需求，默认输出两类脚本：
   - `dev` 联调脚本
   - `prod` 正式上库脚本

## SQL 目录约定

- 开发联调 SQL：`sql/dev/`
- 正式上库 SQL：`sql/prod/`

推荐命名方式：

- `sql/dev/20260402_xxx_dev.sql`
- `sql/prod/20260402_xxx_prod.sql`

## 执行责任

- 开发阶段：允许在 `ruoyi_ai_dev` 中反复修改、验证、回滚。
- 正式阶段：由人工审核后，将 `sql/prod/` 下脚本执行到 `ruoyi`。

## 说明

本约定用于后续所有智能问答模块以及相关数据库改造，除非明确说明，否则默认遵守本流程。
